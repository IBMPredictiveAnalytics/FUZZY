Cette procédure met en correspondance les observations (demandeurs) et les contrôles (fournisseurs) contenus dans un jeu de données unique. La correspondance est basée sur l'égalité exacte ou approximative d'une ou plusieurs variables de correspondance.

Pour chaque observation du groupe demandeur ou d'observations, une tentative de correspondance est effectuée avec les observations du groupe fournisseur ou de contrôle. Lorsque plusieurs correspondances sont demandées, l'ID de l'observation ou des observations fournisseur sélectionnées est écrit sous la forme d'une ou plusieurs nouvelles variables.

Variables de correspondance : Sélectionnez une ou plusieurs variables pour la correspondance. Il n'est pas nécessaire de trier le jeu de données en fonction de ces variables.

Tolérances de correspondance : Si les variables de correspondance relatives aux observations de demandeur et de fournisseur doivent être rigoureusement égales, laissez ce contrôle vide. Si un certain degré de tolérance est requis, indiquez la valeur de tolérance pour chaque variable de correspondance. Indiquez zéro pour une correspondance exacte. Vous pouvez entrer autant de valeurs, séparées par un espace, qu'il y a de variables de correspondance.

• Les variables de chaîne doivent avoir une valeur de tolérance égale à zéro.

Indicateur de groupe : Indiquez la variable qui différencie les observations de demandeur et de fournisseur. Une valeur 1 indique une observation de demandeur et une valeur 0 une observation de fournisseur. Si toute autre valeur ou aucune valeur n'est indiquée, l'observation est ignorée.

ID d'observation : Indiquez une variable d'identificateur d'observation. La valeur d'ID d'observation du ou des fournisseurs sélectionnés sera enregistrée avec chaque observation de demandeur.

Nom des variables ID de correspondance : Indiquez un nom pour la variable qui renfermera l'ID d'observation du fournisseur sélectionné. Vous pouvez entrer plusieurs noms, séparés par un espace. Pour chaque demandeur, le nombre d'observations de fournisseur est déterminé par le nombre de noms de variable indiqué ici. Si aucune correspondance ne peut être générée, la valeur sera manquante par défaut.

• Les noms de variable répertoriés ne doivent pas déjà être présents dans le jeu de données.

Nom de variable de groupe de correspondance : Entrez un nom pour la variable de groupe de correspondance. Cette variable comporte un code indiquant l'ensemble d'observations fournisseur éligibles. La valeur n'est pas significative en elle-même mais deux observations de même valeur ont le même ensemble s'observations fournisseur éligibles.

• Le nom de variable répertorié ne doit pas déjà être présent dans le jeu de données.

Options

Variable du nombre d'observations éligibles : Facultativement, entrez un nom de variable pour enregistrer la taille du pool, par exemple, le nombre d'observations éligibles, pour chaque observation demandeur. Il s'agit du nombre d'observations conformes aux critères de correspondance une fois les correspondances des observations précédentes supprimées, lorsque l'échantillonnage s'effectue sans remplacement.

• Le nom de variable répertorié ne doit pas déjà être présent dans le jeu de données.

Echantillonnage : Les observations fournisseur peuvent être réalisées avec ou sans remplacement. Indiquez l'option voulue.

Priorité aux correspondances exactes : Sélectionnez cette case à cocher pour que l'algorithme de correspondance recherche une correspondance exacte avant une correspondance floue.

Optimisation des performances d'exécution : Sélectionnez cette case à cocher pour optimiser les performances d'exécution avec une vitesse maximale et une utilisation minimale de la mémoire. La sélection de cette case à cocher risque d'augmenter légèrement le nombre d'observations sans correspondance, mais pour les jeux de données volumineux l'amélioration des performances peut être très importante. Certaines statistiques de sortie ne s'appliquent pas, car la procédure n'accumule pas de données sur toutes les correspondances possibles.

• Vous ne pouvez pas indiquer à la fois une priorité exacte et une utilisation minimale de la mémoire.

Organisation aléatoire des observations lors de la création des correspondances : Sélectionnez cette case à cocher afin de créer des correspondances pour le demandeur selon un ordre aléatoire. Sinon, les observations du demandeur sont traitées dans l'ordre où elles se présentent dans le fichier. Cela peut avoir pour conséquence qu'il y aura ensuite moins d'observations à partir desquelles effectuer la sélection dans le fichier. Si les observations du demandeur suivent un ordre rigoureux, cela peut créer une distorsion.

Valeur de départ de nombre aléatoire : Si vous voulez reproduire exactement les mêmes correspondances lors d'une exécution ultérieure avec les mêmes données, vous pouvez définir une valeur de départ pour les nombres aléatoires utilisés lors de la sélection aléatoire dans un ensemble. Cette valeur de départ est sans rapport avec la valeur de départ de nombre aléatoire SPSS Statistics générale.

Sortie supplémentaire

Jeu de données de sortie supplémentaire : Sélectionnez cette case à cocher et entrez un nom de jeu de données en vue de créer un jeu de données renfermant les correspondances. Le nouveau jeu de données contiendra toutes les variables pour chaque observation de fournisseur affectée à une observation de demandeur, suivies de la variable de groupe de correspondance et l'ID de l'observation de demandeur à laquelle elle a été affectée.

• Si vous utilisez cette option une seule observation de fournisseur peut être affectée à chaque observation de demandeur. Par conséquent, la zone Nom des variables ID de correspondance de la boîte de dialogue principale peut contenir un seul nom.

Fonctions supplémentaires

Cette boîte de dialogue génère une syntaxe pour la commande d'extension FUZZY. Pour afficher l'aide sur la syntaxe de cette commande, exécutez la syntaxe suivante :

FUZZY /HELP.

FUZZY comporte un certain nombre de fonctions non prises en charge dans cette boîte de dialogue, notamment l'utilisation de jeux de données d'observation et de contrôle distincts comme entrée. Les fonctions personnalisées écrites dans Python peuvent également être utilisées pour calculer les distances floues. Dans la syntaxe vous pouvez créer un fichier journal qui enregistre la progression au fur et à mesure de l'exécution de la procédure. Cela peut se révéler très utile si vous rencontrez des problèmes importants.

Conditions requises

Cette commande requiert Python Essentials et au moins la version 18 d'IBM SPSS Statistics. L'utilisation d'un jeu de données unique peut nécessiter Statistics version 21 ou un correctif logiciel pour la version 20. Vous pouvez télécharger Python Essentials à partir du site Web de la communauté SPSS à l'adresse www.ibm.com/developerworks/spssdevcentral.
