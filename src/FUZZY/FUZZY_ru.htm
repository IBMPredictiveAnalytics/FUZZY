В этой процедуре сопоставляются наблюдения (группы, выдвигающей требование) и контрольные значения (группы-поставщика), содержащиеся в одном наборе данных. Сопоставление основано на нестрогом или строгом равенстве одной или нескольких переменных сопоставления.

Для каждого наблюдения в группе, выдвигающей требование, или группе наблюдений предпринимается попытка сопоставления с наблюдениями в группе-поставщике или контрольной группе. ID выбранного наблюдения группы-поставщика или нескольких наблюдений, если затребовано несколько сопоставлений, записывается как одна или несколько новых переменных.

Переменные соответствия Выберите одну или более переменных для сопоставления. Сортировка набора данных по этим переменным не требуется.

Допуски соответствия Если переменные сопоставления для наблюдений группы, выдвигающей требование, и наблюдений группы-поставщика должны быть в точности равны, оставьте это контрольное значение пустым. Если нужен какой-либо допуск или нечеткость, введите значение допуска для каждой из сопоставляемых переменных. Для точного сопоставления используйте нулевое значение. Нужно ввести столько значений (через пробел), сколько выбрано переменных сопоставления.

• У строковых переменных должно быть нулевое значение допуска.

Индикатор группы Задайте переменную, различающую наблюдения группы, выдвигающей требования, и группы-поставщика. Значение 1 указывает на наблюдение группы, выдвигающей требование, а значение 0 - на наблюдение группы-поставщика. Все остальные значения, включая пропущенные, приводят к игнорированию наблюдения.

ID наблюдения Задайте переменную идентификатора наблюдения. Значение ID наблюдения для выбранной группы-поставщика, будет храниться с каждым наблюдением группы, выдвигающей требование.

Имена переменных ID сопоставления Задайте имя для переменной, которая будет хранить ID выбранного наблюдения группы-поставщика. Можно ввести несколько имен, разделив их пробелами. Число наблюдений группы-поставщика для каждой группы, выдвигающей требование, определяется числом введенных здесь имен переменных. Если не удастся сгенерировать сопоставление, значение будет системно-пропущенным.

• Имена переменных в списке еще не должны существовать в наборе данных.

Имя группы сопоставления Введите имя для переменной группы сопоставления. Эта переменная будет хранить код, указывающий на набор подходящих наблюдений группы-поставщика. Значение само по себе не важно, но у двух наблюдений с одинаковым значением будет один и тот же набор подходящих наблюдений группы-поставщика.

• Имя переменной в списке должно еще не существовать в наборе данных.

Параметры

Переменная для количества подходящих наблюдений Дополнительно можно ввести имя переменной для записи размера пула (то есть, количества подходящих наблюдений) для каждого наблюдения группы, выдвигающей требование. Это число наблюдений, соответствующих критериям сопоставления после удаления совпадений для предыдущих наблюдений (в случае отбора без возвращения).

• Имя переменной в списке должно еще не существовать в наборе данных.

Отбор Наблюдения группы-поставщика могут быть построены с возвращением или без такового. Укажите нужный вариант.

Отдать приоритет точным соответствиям Включите этот переключатель, чтобы алгоритм сначала пробовал проводить точное сопоставление, а затем нечеткое.

Максимизировать производительность выполнения Включите этот переключатель, чтобы максимизировать производительность выполнения (максимальная скорость и минимальное использование памяти). Включение этого переключателя может немного увеличить число несопоставленных наблюдений, но для больших наборов данных улучшение производительности может оказаться существенным. Некоторые статистики вывода не применяются, поскольку процедура не накапливает данные по всем возможным сопоставлениям.

• Нельзя задать сразу и точный приоритет, и минимизацию памяти.

Рандомизировать порядок наблюдений при построении сопоставлений Включите этот переключатель для построения сопоставлений наблюдений группы, выдвигающей требование, в случайном порядке. В противном случае наблюдения выдвигающей требование группы будут обрабатываться в порядке, соответствующем их порядку в файле. Это может привести к выбору в дальнейшем меньшего числа наблюдений в файле. Если у наблюдений группы, выдвигающей требование, будет систематический порядок, может быть введено некоторое смещение.

Начальное значение генератора случайных чисел Если вы хотите получить возможность точного воспроизведения сопоставления для будущего запуска с теми же самыми данными, можно задать начальное значение для случайных чисел, используемых при отборе из набора случайным образом. Это начальное значение не связано с общим начальным значением генератора случайных чисел SPSS Statistics.

Дополнительный выход

Дополнительный выходной набор данных Включите этот переключатель и введите имя набора данных, чтобы создать новый набор данных, содержащий сопоставления. Новый набор данных будет содержать переменные для каждого наблюдения группы-поставщика, назначенного для наблюдения группы, выдвигающей требование, за которым будет следовать переменная группы сопоставления и ID наблюдения группы, выдвигающей требование, для которого оно было назначено.

• При использовании этой опции для каждого наблюдения группы, выдвигающей требование, может быть назначено только одно наблюдение группы-поставщика. Это значит, что поле Имена переменных ID сопоставления в главном диалоговом окне может содержать только одно имя.

Дополнительные возможности

Это диалоговое окно формирует синтаксис для команды расширения FUZZY. Чтобы показать справку синтаксиса, выполните следующий синтаксис:

FUZZY /HELP.

У команды FUZZY есть ряд возможностей, не поддерживаемых в этом диалоговом окне, включая использование отдельных наборов данных наблюдений и контрольных значений в качестве входных данных. Для вычисления нечеткого расстояния можно также использовать пользовательские функции, написанные на языке Python. В синтаксисе можно создать файл журнала, в котором будет записываться ход выполнения запущенной процедуры. Это может оказаться весьма полезным в случае больших проблем.
