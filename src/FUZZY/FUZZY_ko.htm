이 프로시저는 단일 데이터 세트에 들어 있는 케이스(요구자) 및 제어(공급자)를 매치시킵니다. 매칭은 하나 이상의 매치 변수에 대한 근사 또는 정확한 등식을 기초로 합니다.

요구자 또는 케이스 그룹의 각 케이스에 대해 공급자 또는 대조군의 케이스를 매치시키려 합니다. 선택한 공급자의 ID 또는 ID들(여러 매치가 요청된 경우)은 하나 이상의 새 변수로 기록됩니다.

매치시킬 변수 매치시킬 하나 이사의 변수를 선택하십시오. 데이터 세트는 이 변수별로 정렬하지 않아도 됩니다.

매치 공차 요구자 및 공급자 케이스의 변수 매치가 정확히 동일해야 하는 경우, 이 제어는 비워 두십시오. 일부 공차 또는 "fuzz"를 원하는 경우, 각 매칭 변수에 대해 공차 값을 입력하십시오. 정확히 매치의 경우 0을 입력하십시오. 가능한 많은 값을 입력해야 하며, 매치 변수이므로 공백으로 구분해야 합니다.

• 문자변수의 공차 값은 0이어야 합니다.

집단 지표 요구자와 공급자 케이스를 구분하는 변수를 지정하십시오. 1 값은 요구자 케이스를 나타내고, 0 값은 공급자를 나타냅니다. 결측값을 포함하여 기타 값이 있으면 케이스는 무시합니다.

케이스 ID 케이스 식별자 변수를 지정합니다. 선택한 공급자의 케이스 ID 값은 각 요구자 케이스와 함께 저장됩니다.

매치 ID 변수 이름 선택한 공급자 케이스의 ID가 들어 있는 변수 이름을 지정하십시오. 여러 이름을 공백으로 구분하여 입력할 수 있습니다. 각 요구자에 대한 공급자 케이스 숫자는 여기에 입력한 변수 이름의 갯수로 결정됩니다. 매치를 생성할 수 없는 경우 값은 시스템 결측값이 됩니다.

• 표시된 변수 이름은 아직 데이터 세트에 없는 이름이어야 합니다.

매치 집단 변수 이름 매치 집단 변수의 이름을 입력하십시오. 이 변수에는 적합한 공급자 케이스 세트를 나타내는 코드가 들어 있습니다. 값 자체는 의미가 없지만, 같은 값을 가진 두 케이스에는 적합한 동일 공급자 케이스 세트가 있습니다.

• 표시된 변수 이름은 데이터 세트에 아직 없어야 합니다.

옵션

적합한 케이스 수 변수 선택적으로 각 요구자 케이스에 대해 적합한 케이스 수와 같이 풀의 크기를 기록하는 변수 이름을 입력하십시오. 대체하지 않는 샘플링의 경우 이전 케이스에 대한 매치를 제거한 후 매칭 기준에 맞는 케이스 수입니다.

• 표시된 변수 이름은 데이터 세트에 아직 없어야 합니다.

샘플링 공급자 케이스는 대체를 수행하거나 대체하지 않고 그릴 수 있습니다. 원하는 선택사항을 표시합니다.

정확한 매치에 우선순위 부여 매치 알고리즘이 퍼지 매치를 시도하기 전에 먼저 정확히 매치를 수행하도록 하려면 이 상자를 선택하십시오.

실행 성능 최대화 실행 성능을 최대화하려면 즉 속도는 높이고 메모리 사용량을 줄이려면 이 상자를 선택하십시오. 이 상자를 선택하면 매치하지 않는 케이스 수가 약간 늘어날 수 있지만, 대용량 데이터 세트의 경우 성능이 현저히 개선됩니다. 프로시저가 가능한 모든 매치에서 데이터를 누적하지 않으므로 일부 결과 통계는 적용되지 않습니다.

• 정확한 우선순위와 최소 메모리 모두를 지정할 수 없습니다.

매치할 때 케이스 순서를 무작위화함 랜덤 순서로 요구자 케이스의 매치를 그리려면 이 상자를 선택하십시오. 그렇지 않으면 요구자 케이스가 파일에서 발생하는 순서 대로 처리됩니다. 그러면 적게 발생하는 케이스가 파일에서 나중에 선택됩니다. 요구자 케이스에 시스템에서 지정한 순서가 있는 경우 몇 가지 편향을 도입할 수 있습니다.

난수 시작값 동일한 데이터에 대해 향후 실행 시 정확한 매치를 다시 생성할 수 있도록 하려면, 랜덤으로 세트에서 선택 시 사용되는 난수에 대해 난수 시작값을 설정할 수 있습니다. 이 난수 시작값은 일반 SPSS Statistics 난수 시작값과 관계가 없습니다.

추가 결과

추가 결과 데이터 세트 매치를 포함하는 새 데이터 세트를 작성하려면 이 상자를 선택하고 데이터 세트 이름을 입력하십시오. 새 데이터 세트에는 뒤에 매치집단 변수가 오는 요구자 케이스에 지정된 각 공급자 케이스의 모든 변수 및 지정된 요구자 케이스의 ID가 들어 있습니다.

• 이 옵션을 사용하는 경우 각 요구자 케이스에 대해 하나의 공급자 케이스만 지정될 수 있습니다. 즉 기본 대화 상자에 있는 매치 ID 변수 이름 필드에는 하나의 이름만 포함될 수 있습니다.
