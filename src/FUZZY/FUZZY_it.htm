Questa procedura mette in corrispondenza i casi (richiedenti) e i controlli (fornitori) contenuti in un singolo dataset. La corrispondenza è basata su un'eguaglianza approssimativa o esatta di una o più variabili di corrispondenza.

Per ogni caso nel gruppo di richiedenti o casi, viene tentata una messa in corrispondenza con i casi nel gruppo di fornitori o controlli. L'ID del caso fornitore selezionato o dei casi, se vengono richieste più corrispondenze, viene scritto come una o più nuove variabili.

Variabili in base a cui eseguire la messa in corrispondenza Selezionare una o più variabili in base a cui eseguire la messa in corrispondenza. Il dataset non deve essere ordinato in base a queste variabili.

Tolleranza delle corrispondenze Se le variabili di corrispondenza per i casi richiedente e fornitore devono essere esattamente uguali, lasciare questo controllo vuoto. Se è desiderata una certa tolleranza o "approssimazione", immettere il valore di tolleranza per ogni variabile corrispondente. Utilizzare zero per una corrispondenza esatta. Il numero di valori immessi, separati da uno spazio, deve essere uguale a quello delle variabili di corrispondenza.

• Le variabili stringa devono avere un valore di tolleranza pari a zero.

Indicatore gruppo Specificare la variabile che distingue i casi richiedente e fornitore. Il valore 1 indica un caso richiedente e il valore 0 indica un fornitore. Per gli altri valori, compresi quelli mancanti, il caso verrà ignorato.

ID caso Specificare una variabile identificativo del caso. Il valore ID caso per il fornitore o i fornitori selezionati sarà archiviato con ogni caso richiedente.

Nomi per variabili ID corrispondenza Specificare un nome per la variabile che conterrà l'ID del caso fornitore selezionato. È possibile immettere più nomi, separati da uno spazio. Il numero di casi fornitore per ogni richiedente è determinato dal numero di nomi variabile immesso qui. Se non è possibile produrre una corrispondenza, il valore sarà mancante di sistema.

• I nomi variabile elencati non devono esistere già nel dataset.

Nome della variabile di gruppo corrispondenza Immettere il nome per la variabile di gruppo corrispondenza. Questa variabile conterrà un codice che indica la serie di casi fornitore idonei. Il valore stesso non è significativo, ma due casi con lo stesso valore hanno la stessa serie di casi fornitore idonei.

• Il nome variabile elencato non deve esistere già nel dataset.

Opzioni

Variabile per numero di casi idonei Facoltativamente, immettere un nome di variabile per registrare la dimensione del pool, ossia il numero di casi idonei, per ogni caso richiedente. Questo è il numero di casi che soddisfa i criteri di messa a confronto dopo che le corrispondenze per i casi precedenti sono state rimosse, in caso di campionamento senza sostituzione.

• Il nome variabile elencato non deve esistere già nel dataset.

Campionamento I casi fornitore possono essere estratti con o senza sostituzione. Indicare la scelta desiderata.

Dai la priorità alle corrispondenze esatte Selezionare questa casella per fare in modo che l'algoritmo di messa in corrispondenza provi prima una corrispondenza esatta per poi tentare una corrispondenza approssimativa.

Massimizza prestazioni esecuzione Selezionare questa casella per massimizzare le prestazioni dell'esecuzione - massima velocità e minimo utilizzo della memoria. La selezione di questa opzione potrebbe aumentare leggermente il numero di casi senza corrispondenza ma, per i dataset di grandi dimensioni, il miglioramento delle prestazioni può essere considerevole. Alcune delle statistiche di output non si applicano perché la procedura non accumula dati su tutte le corrispondenze possibili.

• Non è possibile specificare sia la priorità esatta che la minimizzazione della memoria.

Rendi casuale l'ordine dei casi quando si estraggono le corrispondenze Selezionare questa casella per estrarre le corrispondenze per i casi richiedente in ordine casuale. Altrimenti, i casi richiedente vengono elaborati nell'ordine in cui ricorrono nel file. Ciò potrebbe determinare un numero di casi inferiore da cui effettuare la selezione successivamente nel file. Se i casi richiedenti hanno un ordine sistematico, questo potrebbe introdurre un certo grado di distorsione.

Seme per numeri casuali Se si desidera poter riprodurre le corrispondenze esattamente in una futura esecuzione con gli stessi dati, è possibile impostare un valore del seme per i numeri casuali utilizzati quando si effettua la selezione da un insieme in modo casuale. Questo seme non è correlato al seme per numeri casuali generale di SPSS Statistics.

Output aggiuntivo

Dataset di output aggiuntivo Selezionare la casella e immettere un nome di dataset per creare un nuovo dataset contenente le corrispondenze. Il nuovo dataset conterrà tutte le variabili per ogni caso fornitore che è stato assegnato a un caso richiedente seguito dalla variabile di gruppo di corrispondenza e dall'ID del caso richiedente a cui è stato assegnato.

• Se si utilizza questa opzione, a ogni caso richiedente è possibile assegnare un solo caso fornitore. Ciò significa che il campo Nomi per variabili ID di corrispondenza nella finestra di dialogo principale può contenere un solo nome.

Funzioni aggiuntive

Questa finestra di dialogo genera la sintassi per il comando di estensione FUZZY. Per visualizzare la guida alla sintassi per questo comando, eseguire la seguente sintassi:

FUZZY /HELP.

FUZZY presenta diverse funzioni non supportate in questa finestra di dialogo, compresi l'utilizzo di dataset di controlli e casi separati come input. Le funzioni personalizzate scritte in Python possono essere utilizzate anche per calcolare la distanza approssimativa. Nella sintassi, è possibile creare un file di log che registra l'avanzamento man mano che la procedura viene eseguita. Ciò può essere molto utile nei problemi di grandi dimensioni.

Requisiti

Questo comando richiede Python Essentials e almeno la versione 18 di IBM SPSS Statistics. Per l'utilizzo di dataset singoli, potrebbe essere richiesto Statistics versione 21 oppure una hot fix per la versione 20. È possibile scaricare Python Essentials dal sito Web di SPSS Community all'indirizzo www.ibm.com/developerworks/spssdevcentral.
